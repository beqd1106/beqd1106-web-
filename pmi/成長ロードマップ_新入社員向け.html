04 スキルマップ詳細

具体的に何をどのレベルまで習得するか

▶が目標レベル（1〜3年での達成イメージ）

● = 習得済み ○ = 目標
財務分析
目標：中小企業の決算書を見て、健全性・収益性・PMI後のリスクを口頭で説明できる
ステップ：BS/PL/CF読解 → 収益性分析 → 企業価値評価の基礎（EV/EBITDA等）
ロジカルシンキング
目標：顧客課題をMECEで構造化し、ロジックツリーで解決策を提案できる
ステップ：MECE基礎 → ロジックツリー → ピラミッドストラクチャーでの提案書作成
PMI・M&A知識
目標：M&AのDay1からDay100まで、フェーズごとに何をすべきかを説明し実行できる
ステップ：M&A基本フロー → DD基礎 → 100日プラン設計 → シナジー追跡 → 組織融合
業務フロー設計
目標：AS-IS（現状）とTO-BE（理想）の業務フローをBPMN/スイムレーン図で作成できる
ステップ：フロー図ツール習得（Draw.io等）→ AS-IS分析 → TO-BE設計 → 実装支援
変革マネジメント
目標：ADKARモデルを使って、M&A後の組織変革に対する人材の抵抗を低減できる
ステップ：ADKAR理論理解 → ステークホルダー分析 → コミュニケーション計画 → Prosci認定
AIプロンプト設計
目標：PMI Manager上でAI機能（100日プラン提案・リスク検知）のプロンプトを設計・改善できる
ステップ：プロンプトエンジニアリング基礎 → Claude API実装 → RAG活用 → 品質評価
スライド・資料作成
目標：経営層・投資家・顧客それぞれに合わせたプレゼンテーション資料を単独で作れる
ステップ：パワポ構成力 → ビジュアル表現 → エグゼクティブ向けサマリー → ピッチ資料

●の数は1〜2年での習得目標レベル（5段階中）。入社時は全て0からスタートで構わない。

05 必修フレームワーク

コンサルタントが使うフレームワーク

まずはBasicを全部使いこなし、Middleへ。Advancedは実案件で自然に身につく

Basic

MECE

「モレなく・ダブりなく」で課題を整理する。全ての分析の土台。最初に習得すべき思考ツール。

Basic

ロジックツリー

問題を原因ツリー・解決策ツリーに分解。MECEと合わせて使う。提案書の骨格になる。

Basic

3C分析

Customer（顧客）・Competitor（競合）・Company（自社）の3軸で市場を分析する。

Basic

SWOT分析

強み・弱み・機会・脅威の4象限。クロスSWOTまで使いこなせるとプロ級。

Basic

PDCA

Plan-Do-Check-Actの改善サイクル。PMI後の運用フェーズで活用頻度が高い。

Middle

PEST分析

Political・Economic・Social・Technologicalで外部環境を分析。M&A時の市場調査で使う。

Middle

バリューチェーン分析

企業活動を主活動・支援活動に分解。PMIでシナジーポイントを特定する際に必須。

Middle

ADKAR モデル

Awareness-Desire-Knowledge-Ability-Reinforcement。組織変革・文化融合の変革マネジメント手法。

Middle

ガントチャート

プロジェクト工程管理。100日プランの可視化・進捗追跡に直接使う。PMI Manager内でも中核機能。

Advanced

5フォース分析

業界の競争環境を5要因で分析（Porter）。買収対象企業の業界分析で使用。

Advanced

DCF法（企業価値評価）

将来キャッシュフロー割引による企業価値算定。DD・買収価格交渉で必要。

Advanced

OKR（目標管理）

Objectives & Key Results。PMI後のシナジー目標設定と組織目標管理に活用できる。

08 週次ルーティン

最初の3ヶ月の「1週間の過ごし方」

毎日コツコツ積み上げる習慣が、1年後に大きな差になる

MON

学習インプット

書籍・テキスト30分読書。今週学ぶテーマを決めて、ToDoに落とす。

TUE

フレームワーク練習

ニュースの企業ネタを1つ選び、3C分析orSWOTを手書きで作る（15分）。

WED

AI実験・プロト

Claude/ChatGPTに業務プロンプトを1つ試す。使えそうな発見を社内共有。

THU

資格勉強

進行中の資格テキストを1時間。過去問を5問解いて解説を確認。

FRI

振り返り・アウトプット

今週学んだことを200字でまとめてNotionに記録。来週の学習テーマを設定。

SAT

M&A・業界情報収集

日経・M&Aニュースを30分読む。事業承継・中小企業の最新動向を把握。

SUN

完全オフ or 読書

無理に勉強しない。ただし興味が出た本は気軽に開く。燃え尽きないことが大切。

累積学習時間の目安：上記ルーティンを守ると月約40〜50時間の学習時間が確保できる。1年で約500時間。これがあればITパスポート・事業承継エキスパート・基本情報技術者の3資格は十分取得可能。
09 マインドセット

当社流「ラストマン精神」の実践

スキルより先に、この会社のカルチャーを体に入れる

🔥

ラストマン精神 ― 「私がやる」が最初の一言

「誰かがやるだろう」はない。議事録、雑用、顧客フォロー、なんでも「私がやります」と手を挙げる。最初のうちは量をこなすことが質への近道。

🏗️

現場主義 ― 顧客の「現場」に足を運ぶ

資料の中にない情報が現場にある。PMIの本質は「人」と「現場」。顧客の工場・店舗・オフィスに実際に行き、目で見て感じる習慣を作る。

📊

データ駆動 ― 感覚より数字で話す

「なんとなく課題があります」ではなく「〇〇の作業が週に△時間かかっており、年間□万円のコストになっています」と数字で語る。AIを使えば数字収集も速くなる。

🚀

「量」の先に「質」がある ― 最初から完璧を求めない

粗くてもいいから毎日アウトプットを出す。フレームワークを100回使えば自然と使いこなせる。資料作成も100枚作れば1枚が速くなる。まず量をこなす。

🤝

ホスピタリティ ― コンサルの本質は「相手への贈り物」

「本当のコンサルティングとはホスピタリティ」という会社の言葉を胸に刻む。顧客が何に困っていて、何を本当に求めているかを徹底的に想像する。技術より先に人への興味を持つ。
